# marble-solitaire-model

AlphaZero-style neural network that learns to solve European 37-hole marble solitaire through self-play.

## Architecture
- **Board**: European 37-hole, represented as 7x7 grid with validity mask
- **Model**: Dual-headed CNN (configurable: 75K-450K params via `--channels`/`--n-blocks`) — policy (196 move logits) + value (scalar)
- **Training**: Single-player MCTS with Dirichlet noise → self-play → replay buffer → SGD. Optional bootstrap from prior checkpoint and seed-search at startup.
- **Inference**: Greedy policy from ONNX model in browser via onnxruntime-web
- **Web UI**: Vanilla TypeScript + Vite, SVG board, 5 generation snapshots showing learning progression

## Key Files
- `src/marble_solitaire/board.py` — board state, move generation, move encoding
- `src/marble_solitaire/model.py` — SolitaireNet (configurable channels + blocks)
- `src/marble_solitaire/mcts.py` — single-player MCTS, Dirichlet noise, `compute_outcome` rewards
- `src/marble_solitaire/self_play.py` — episode generation, replay buffer, temperature schedule
- `src/marble_solitaire/train.py` — training loop with checkpoint resume
- `src/marble_solitaire/inference.py` — greedy policy solver (device-aware)
- `src/marble_solitaire/export.py` — ONNX export for browser inference
- `src/marble_solitaire/seeds.py` — generate seed solutions via high-MCTS search
- `web/src/solver.ts` — browser-side ONNX inference + move masking
- `web/src/board.ts` — SVG rendering + animation
- `vertex-training/` — Vertex AI Custom Training (GPU)
  - `Dockerfile` — PyTorch 2.6 + CUDA 12.4, non-root user
  - `run.py` — training entrypoint with bootstrap + seed options
  - `cloudbuild.yaml` — Cloud Build config (uses `cloudbuild-deploy@` SA)
  - `phase2-spec.yaml` — Phase 2 job config

## Move Encoding (CRITICAL)
Moves are encoded as `(row * 7 + col) * 4 + direction` where direction is UP=0, DOWN=1, LEFT=2, RIGHT=3. This encoding MUST match between Python (`board.py`) and TypeScript (`solver.ts`). A mismatch here causes the browser to play completely wrong moves.

## Reward Function (compute_outcome)
- `+1.0` — 1 marble at center (perfect solve)
- `+0.6` — 1 marble off-center
- `-0.3` — 2 marbles remaining
- `-1 + 2/n` — smooth gradient for n ≥ 3 marbles

The gap between "+1.0 center solve" and "-0.3 two marbles" is what gives training the signal to push for full solves.

## Training (Cloud only)
The dev VM has no torch installed. All training runs on Vertex AI.

```bash
# Build the training container (writes to gcr.io with timestamped tag)
./vertex-training/build.sh

# Submit a job
gcloud ai custom-jobs create --region=us-central1 \
  --display-name=marble-solitaire-phase2 \
  --config=vertex-training/phase2-spec.yaml \
  --project=ai-dev-463705
```

Job arguments (passed via `args:` in spec):
- `--iterations`, `--episodes`, `--simulations` — training scale
- `--channels`, `--n-blocks` — network size
- `--checkpoint-iters` — comma-separated iters that save .pt files
- `--bootstrap-checkpoint` — gs:// path to a `.pt` to load before training (must match `--channels`/`--n-blocks`)
- `--seed-attempts`, `--seed-sims`, `--seed-copies` — seed-search options at startup

Outputs go to `gs://ai-dev-463705-ml-artifacts/marble-solitaire/<run>/` (set via `AIP_MODEL_DIR` env var in spec).

## Deploying Updated Models
```bash
# Download ONNX models from the completed Vertex run
gsutil -m cp gs://ai-dev-463705-ml-artifacts/marble-solitaire/<run>/onnx/*.onnx web/public/models/

# Rebuild dist with new models
cd web && npx vite build

# Commit web/dist/ in monorepo root and deploy via /go
```

## Rules
- Move encoding must stay in sync between Python and TypeScript — test both when changing
- web/dist/ is checked into git (needed for Docker deploy) — rebuild before committing
- Model artifacts (.pt, .onnx) in models/ are gitignored — regenerated by training
- Always use red/green TDD for new features and bug fixes
- All GCP admin (IAM, quotas, APIs) must be done via Console — the VM's compute SA is intentionally minimal
